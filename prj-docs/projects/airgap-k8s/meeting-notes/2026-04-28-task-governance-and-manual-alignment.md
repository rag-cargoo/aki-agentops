# Meeting Notes: Task Governance and Manual Alignment

## 안건 1: `task.md`에 어떤 TODO를 적을지 기준 정리
- Created At: 2026-04-28 01:35:00
- Updated At: 2026-04-28 01:35:00
- Status: DONE
- 검토사항:
  - 예상 단계나 아직 합의되지 않은 작업까지 `task.md`에 미리 적어두면 실제 진행 상황과 섞여 보인다.
  - 과제 진행판은 현재 확정된 일과 완료된 일을 빠르게 읽을 수 있어야 한다.
- 결정사항:
  - `task.md`에는 `확정된 절차`와 `실제로 시작했거나 바로 시작할 작업`만 기록한다.
  - 아직 결정되지 않은 예상 단계는 `meeting-notes/*.md` 또는 `manual/00-제출-매뉴얼-개요.md`에만 남기고 `task.md`에는 올리지 않는다.
  - TODO는 하나의 완료 기준이 분명한 작업 단위로만 적는다.

## 안건 2: `task.md`를 제출 매뉴얼 순서에 맞춰 세분화할지 판단
- Created At: 2026-04-28 01:35:00
- Updated At: 2026-04-28 01:35:00
- Status: DONE
- 검토사항:
  - 과제는 단계별 매뉴얼 제출이 요구되므로 진행판도 같은 순서로 정렬되는 편이 낫다.
  - 다만 모든 장을 미리 TODO 섹션으로 다 만들어 두면 다시 지저분해질 수 있다.
- 결정사항:
  - `task.md`의 `Current Items`는 실제 제출 매뉴얼 장 순서에 맞춰 묶는다.
  - 항목이 실제로 확정된 장만 표시하고, 아직 작업이 없는 장은 생략한다.
  - `Next Items`는 예측 목록으로 쓰지 않고, 현재 확정되어 있지만 아직 착수 전인 항목이 있을 때만 사용한다.
