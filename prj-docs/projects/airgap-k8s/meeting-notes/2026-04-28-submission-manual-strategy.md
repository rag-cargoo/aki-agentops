## 안건 1: 제출용 매뉴얼 원본 형식 결정
- Created At: 2026-04-28 01:20:00
- Updated At: 2026-04-28 01:20:00
- Status: DONE
- 검토사항:
  - 제출 요구사항은 `서버 구축 과정에 대한 상세한 단계별 매뉴얼 작성본`이며 최종 확장자는 자유다.
  - 작업 이력 관리와 diff 추적, 재작성 속도는 `Markdown`이 가장 유리하다.
  - 최종 제출 시에는 Word/HWP/PPT/PDF로 변환할 수 있어야 한다.
- 결정사항:
  - 제출용 매뉴얼의 단일 원본은 프로젝트 루트 `manual/*.md`로 관리한다.
  - 최종 제출 직전에 필요 형식으로 변환하되, 내용 기준은 `manual/*.md`를 따른다.
  - 진행 중인 설치 명령, 설정 파일 경로, 검증 결과도 먼저 Markdown에 누적한다.
  - `manual/00-제출-매뉴얼-개요.md`는 공통 서문으로 유지한다.
  - 장별 파일은 실제 내용이 생길 때만 추가한다.

## 안건 2: 제출용 매뉴얼 디렉터리 구조 결정
- Created At: 2026-04-28 01:20:00
- Updated At: 2026-04-28 01:20:00
- Status: DONE
- 검토사항:
  - 한 파일에 모든 내용을 밀어 넣으면 수정과 검증이 어렵다.
  - 평가자는 `순서대로 따라 할 수 있는 구성`을 선호할 가능성이 높다.
  - AWS 사용 이유와 폐쇄망 적합성 설명은 앞부분에서 선제적으로 해소해야 한다.
- 결정사항:
  - 프로젝트 루트에 `manual/` 디렉터리를 만들고 번호형 문서로 분리한다.
  - `00`은 공통 개요, `01`부터 실제 절차 매뉴얼로 사용한다.
  - 실제 제출 매뉴얼의 첫 파일은 `manual/02-환경-준비.md`로 시작한다.
  - 번호형 문서는 아래 순서를 기본으로 한다.
    - `01` 인프라 구축
    - `02` 서비스 배포
    - `03` 모니터링 및 외부 접근
    - `04` 검증 및 제출 정리
