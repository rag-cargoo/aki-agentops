## Sidecar Draft
- 외부 접근 방식과 실제 Service/Ingress 파일이 확정되기 전까지 아래 내용은 제출 매뉴얼에 넣지 않는다.
- 실행 후 접근 URL, 포트, 응답 결과를 매뉴얼에 반영한다.

```bash
make 05-prometheus-grafana-external-access-run
make 05-prometheus-grafana-external-access-verify
kubectl -n ingress-nginx get svc ingress-nginx-controller
kubectl -n monitoring get ingress
```

Default hosts:
- `grafana.airgap.local`
- `prometheus.airgap.local`

Default MetalLB address pool:
- `10.10.20.240-10.10.20.250`

## Result
- LoadBalancer Service: `ingress-nginx/ingress-nginx-controller`
- Assigned IP: `10.10.20.240`
- Grafana route: `http://grafana.airgap.local -> 10.10.20.240`
- Prometheus route: `http://prometheus.airgap.local -> 10.10.20.240`
- Verification: `make 05-prometheus-grafana-external-access-verify` success
- Prerequisite note: AWS private subnet simulation requires Calico `VXLAN` encapsulation so kube-apiserver can reach admission webhook pods across nodes.
