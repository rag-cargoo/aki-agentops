## Sidecar Draft
- 실제 클러스터 구성 전까지 아래 내용은 제출 매뉴얼에 넣지 않는다.
- 수동 구성과 Ansible 구성은 둘 다 준비하되, 실행 결과가 생긴 방식만 매뉴얼 본문에 반영한다.
- 이 장은 AWS 인스턴스가 아니라 VM 노드 기준 설치 절차로 진행해도 무방하다.
- 사전점검 스크립트: `ops/03-kubernetes-cluster/scripts/kubeadm-preflight-check.sh`
- 로컬 작업 원본: `assets/offline-assets/`
- 반입 번들 경로: `delivery/offline-assets/`
- 서버 반입 경로: `/opt/offline-assets`
- 서버 설치 명령은 `/opt/offline-assets/...` 기준으로 작성한다.
- bastion 경유 접속 명령은 `manual/03-쿠버네티스-클러스터-구성/README.md` 기준으로 기록한다.
- `nc` 포트 점검 예시는 kubeadm 설치 전 추가 검증으로 `manual/03`에 둔다.

## Execution Order
1. Kubernetes 공식문서 링크를 매뉴얼과 sidecar에 정리한다.
2. `v1.35.3 + containerd + Calico` 기준 다운로드 대상 패키지와 이미지 목록을 확정한다.
3. 로컬 PC에서 실제 다운로드를 수행하고 파일이 정상 수집되는지 확인한다.
4. `delivery/offline-assets/`에 서버 반입용 번들을 정리한다.
5. `offline-assets.tar.gz` 반입 압축 구조를 만든다.
6. VM에 `/opt/offline-assets`로 반입한 뒤 설치를 검증한다.
7. 실제 성공한 명령만 `manual/03-쿠버네티스-클러스터-구성/README.md`에 반영한다.

```bash
rsync -av assets/offline-assets/ delivery/offline-assets/
```

```bash
tar -czf offline-assets.tar.gz -C delivery offline-assets
```

```bash
sudo mkdir -p /opt
sudo tar -xzf offline-assets.tar.gz -C /opt
```

```bash
sudo swapoff -a
sudo sed -i '/ swap / s/^/#/' /etc/fstab
sudo modprobe overlay
sudo modprobe br_netfilter
sudo sysctl --system
```

```bash
sudo mkdir -p /etc/containerd
sudo containerd config default | sudo tee /etc/containerd/config.toml
sudo systemctl restart containerd
sudo systemctl enable containerd
```

```bash
sudo dpkg -i /opt/offline-assets/kubernetes/packages/kubernetes/*.deb
sudo apt-mark hold kubelet kubeadm kubectl
kubeadm config images list --kubernetes-version v1.35.3
```

```bash
sudo kubeadm init \
  --kubernetes-version v1.35.3 \
  --pod-network-cidr <POD_CIDR>

kubeadm token create --print-join-command
kubectl get nodes -o wide
```

ansible-playbook -i inventory.ini playbooks/02-containerd.yml
