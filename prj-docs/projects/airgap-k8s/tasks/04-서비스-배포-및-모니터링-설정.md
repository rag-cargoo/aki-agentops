## Architecture Decision
- `04`는 서비스별 독립 단원으로 나눈다.
- 각 서비스는 사용할지 말지 선택할 수 있어야 하므로 다운로드, 검증, 전송, 이미지 import, 배포, 검증 책임을 서비스 디렉터리 안에 둔다.
- 공통 묶음 target은 선택 편의용으로만 둔다. 실제 책임 단위는 서비스별 target이다.
- 서비스별 자산 목록은 각 서비스의 `assets/images.txt`에서 관리한다.
- 다운로드 결과물은 `assets/offline-assets/services/images/<service>/`와 `assets/offline-assets/services/manifests/<service>/` 아래에 서비스별로 분리한다.
- master에는 manifest를 전달하고, worker에는 서비스 이미지 tar를 전달한다.
- 서비스 이미지 tar는 worker에서 `ctr -n k8s.io images import` 후 삭제해 `DiskPressure`를 방지한다.
- DB와 모니터링 서비스의 PVC는 `03` 단계에서 구성한 기본 StorageClass `local-path`를 사용한다.

## Troubleshooting Recorded
- 8GB EC2 root volume에서 서비스 이미지와 offline tar가 누적되어 `DiskPressure`가 발생했다.
- Terraform에 `node_root_volume_size = 30`을 추가하고 현재 master/worker EBS 볼륨을 30GB로 확장했다.
- 03/04 이미지 import 스크립트는 원격 tar를 import 후 삭제하도록 조정했다.
- 상세 문서: `manual/troubleshooting/04-services-diskpressure-root-volume.md`
