## Architecture Decision
- `04`는 서비스별 독립 단원으로 나눈다.
- 각 서비스는 사용할지 말지 선택할 수 있어야 하므로 다운로드, 검증, 전송, 이미지 import, 배포, 검증 책임을 서비스 디렉터리 안에 둔다.
- 공통 묶음 target은 선택 편의용으로만 둔다. 실제 책임 단위는 서비스별 target이다.
- 서비스별 자산 목록은 각 서비스의 `assets/` 아래에서 관리한다.
- 다운로드 결과물은 `assets/offline-assets/services/<service>/` 아래에 서비스별로 분리한다.
- 서버 반입본은 기존 기준대로 `delivery/offline-assets/`와 `delivery/offline-assets.tar.gz`에 포함한다.
- 제출 매뉴얼에는 실제 manifest/values와 실행 검증이 끝난 뒤 결과 기준으로만 반영한다.
