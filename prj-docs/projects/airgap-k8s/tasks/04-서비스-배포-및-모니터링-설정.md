## Sidecar Draft
- 실제 manifest 또는 Helm values가 작성되기 전까지 아래 내용은 제출 매뉴얼에 넣지 않는다.
- `ops/04-services-monitoring/` 아래 실제 설정 파일이 생긴 뒤 실행 결과 기준으로 매뉴얼에 반영한다.

```bash
kubectl create namespace database
kubectl create namespace monitoring
kubectl apply -f ops/04-services-monitoring/mysql-or-mariadb/
kubectl apply -f ops/04-services-monitoring/mongodb/
kubectl apply -f ops/04-services-monitoring/prometheus/
kubectl apply -f ops/04-services-monitoring/grafana/
kubectl apply -f ops/04-services-monitoring/grafana-alloy/
kubectl get pods -A
kubectl get svc -A
kubectl get pvc -A
```
