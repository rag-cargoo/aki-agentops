# 📝 Technical Documentation Expert

이 스킬은 워크스페이스 내의 모든 기술 문서가 **"엔지니어링 리포트 수준의 전문성"**을 유지하도록 관리하는 에이전트 전용 지침입니다.

## ⚖️ 3대 불변 원칙 (Immutable Rules)

1.  **데이터 무손실 원칙 (Zero-Loss Policy)**: 리팩토링이나 스타일 수정 시, 기존의 상세 텍스트와 코드를 단 한 줄도 요약하거나 삭제할 수 없습니다. 수정 전후 `wc -l` 비교는 필수입니다.
2.  **레이아웃 정규화 (Normalization)**: 모든 문서는 `H2 헤더 -> 구분선 -> 인용구(>) 박스 -> H3 소제목 -> 구분선`으로 이어지는 고정된 시각적 위계를 따라야 합니다.
3.  **색상 블록 시스템 (Callout System)**: 목적에 맞는 Docsify Alert 박스(`[!NOTE]`, `[!TIP]`, `[!WARNING]`)를 사용하여 정보의 성격을 명확히 구분합니다.
