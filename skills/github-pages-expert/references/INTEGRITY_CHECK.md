#  데이터 무손실 보장 수칙 (Integrity Check)

> **Goal**: 리팩토링 중 단 한 줄의 핵심 정보도 유실되지 않도록 물리적 검증을 수행한다.

---

## 1. 수정 전 필수 절차
1.  **라인 수 기록**: `wc -l [파일명]` 명령어로 현재 라인 수를 측정한다.
2.  **백업**: `cp [파일명] [파일명].bak`으로 물리적 복사본을 생성한다.

## 2. 수정 후 검증 절차
1.  **라인 수 대조**: 
    - 정보가 추가되는데 라인 수가 줄어들면 사고로 간주한다.
    - 최소한 원본 라인 수의 95% 이상이 유지되어야 한다. (스타일 최적화 고려)
2.  **핵심 키워드 검색**: 
    - 원본에 있던 주요 코드 블록이나 철학적 키워드가 살아있는지 `grep` 등으로 확인한다.

## 3. 복구 프로토콜
- 검증 실패 시, 어떠한 자기합리화도 하지 말고 즉시 `mv [파일명].bak [파일명]`으로 원상 복구한 뒤 사용자에게 보고한다.
