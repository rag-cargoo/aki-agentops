# 🌐 Workspace Global Governance

이 스킬은 워크스페이스 내 모든 프로젝트의 일관된 품질과 거버넌스를 보장하기 위한 **'에이전트 수칙'**입니다.

## ⚖️ 핵심 5대 원칙 (Core Principles)

1. **데이터 무결성 절대 사수**: 기존 파일을 수정할 때 `write_file` 사용을 금지하고 `replace`를 사용하여 데이터 소실을 원천 차단한다. (상세: `OPERATIONS.md`)
2. **DoD(완료 정의) 패키지 준수**: 기능 구현 시 [코드 수정 + 문서 현행화 + API 명세 갱신 + 테스트 도구 최신화]를 하나의 원자적 작업으로 완수한다. (상세: `WORKFLOW.md`)
2. **하드코딩 절대 금지**: 모든 설정, ID, 경로는 환경 변수나 설정 파일을 통해 주입받는다. (상세: `CODING_STANDARD.md`)
3. **3단 계층 구조 사수**: `api`(인터페이스), `domain`(비즈니스), `global`(기술지원) 패키지 경계를 엄격히 유지한다. (상세: `STRUCTURE.md`)
4. **API 명세 6단계 템플릿**: 모든 API 문서는 표준 템플릿을 반드시 준수한다. (상세: `STRUCTURE.md`)
5. **정석적인 형상 관리**: 커밋 전 문서 정합성을 최종 검토하고, 한글로 명확한 변경 사유를 기록한다. (상세: `WORKFLOW.md`)
6. **스킬 지식 동기화**: 거버넌스 규칙(`skills/`) 수정 시 반드시 `sync-skill.sh`를 통해 런타임(`.gemini/skills/`)에 반영하고 `/skills reload`를 수행한다. (상세: `OPERATIONS.md`)

## 🤖 세션 시작 시 가이드 (Agent Boot Protocol)
새로운 세션이 시작되면 에이전트는 사용자에게 다음 사항을 확인하고 안내해야 합니다.
1.  **지식 동기화 확인**: "스킬 소스와 런타임이 링크되어 있는지 확인하셨나요? 필요한 경우 `scripts/sync-skill.sh`를 실행해 주세요."
2.  **지식 갱신 요청**: "최신 규칙 반영을 위해 `/skills reload` 입력을 부탁드립니다."
3.  **스킬 현황 가시화**: 에이전트는 사용자가 "스킬스 싱크?" 요청 시, 현재 메모리에 로드된 **모든 스킬과 그 버전(2026-02-07 15:30:00)**을 테이블 형식으로 일목요연하게 나열해야 합니다.
4.  **실행 모니터링 (Execution Monitoring)**: 쉘 명령 실행 중 오류가 발생하거나 응답이 비정상적으로 길어질 경우, 에이전트는 즉시 상황을 분석하고 사용자에게 중단 또는 수정 방안을 제안해야 합니다.
5.  **작업 문맥 로드**: 안내 후 즉시 `prj-docs/task.md`를 읽어 중단된 지점부터 이어서 진행합니다.
