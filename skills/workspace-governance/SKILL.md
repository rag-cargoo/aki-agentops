# 🌐 Workspace Global Governance

이 스킬은 워크스페이스 내 모든 프로젝트의 일관된 품질과 거버넌스를 보장하기 위한 **'에이전트 수칙'**입니다.

## ⚖️ 핵심 5대 원칙 (Core Principles)

1. **데이터 무결성 절대 사수**: 기존 파일을 수정할 때 `write_file` 사용을 금지하고 `replace`를 사용하여 데이터 소실을 원천 차단한다. (상세: `OPERATIONS.md`)
2. **문서 전문성 위임**: 모든 기술 문서의 작성, 리팩토링, 시각적 스타일링은 `document-expert` 스킬의 지침을 반드시 따른다.
3. **하드코딩 절대 금지**: 모든 설정, ID, 경로는 환경 변수나 설정 파일을 통해 주입받는다. (상세: `CODING_STANDARD.md`)
4. **3단 계층 구조 사수**: `api`(인터페이스), `domain`(비즈니스), `global`(기술지원) 패키지 경계를 엄격히 유지한다. (상세: `STRUCTURE.md`)
5. **정석적인 형상 관리**: 커밋 전 문서 정합성을 최종 검토하고, 한글로 명확한 변경 사유를 기록한다. (상세: `WORKFLOW.md`)

## 🛠️ 작업 시 필수 참조 (Mandatory Reference)

| 행동 단계 | 반드시 읽어야 할 참조 문서 | 검증 포인트 |
| :--- | :--- | :--- |
| **설계/기획** | `ARCHITECTURE_RULES.md` | 모듈 간 참조 제한 준수 여부 |
| **코드 수정** | `CODING_STANDARD.md` | 하드코딩 유무, 패키지 위치 |
| **문서 작성** | `document-expert` 활성화 | 무손실 복구 및 정규화 스타일 준수 |
| **테스트/운영** | `OPERATIONS.md` | Makefile 등록, 스크립트 작성 규격 |
| **커밋/푸시** | `WORKFLOW.md` | 문서 현행화 및 정합성 최종 체크 |

---
**이 스킬이 활성화된 동안 에이전트는 문서의 '디자인'보다 '거버넌스'에 집중하며, 시각적 품질은 `document-expert`의 전문성을 빌려 해결해야 한다.**

## 🤖 세션 시작 시 가이드 (Agent Boot Protocol)
새로운 세션이 시작되면 에이전트는 사용자에게 다음 사항을 확인하고 안내해야 합니다.
1.  **지식 동기화 확인**: "스킬 소스와 런타임이 링크되어 있는지 확인하셨나요? 필요한 경우 `scripts/sync-skill.sh`를 실행해 주세요."
2.  **지식 갱신 요청**: "최신 규칙 반영을 위해 `/skills reload` 입력을 부탁드립니다."
3.  **스킬 현황 가시화**: 에이전트는 사용자가 "스킬스 싱크?" 요청 시, 현재 메모리에 로드된 **모든 스킬과 그 버전**을 테이블 형식으로 나열해야 합니다.
4.  **실행 모니터링**: 쉘 실행 오류나 지연 발생 시 즉시 보고하고 대안을 제시합니다.
5.  **작업 문맥 로드**: 안내 후 즉시 `prj-docs/task.md`를 읽어 진행합니다.
